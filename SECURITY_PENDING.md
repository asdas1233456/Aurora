# Security Pending Backlog

本文档用于单独记录 Aurora 当前识别到的安全与部署边界问题。

说明：

- 这些事项暂时不纳入当前主产品路线图排序。
- 当前状态统一视为 `Pending`。
- 后续进入内网共享、多人使用或对外部署前，建议优先回收本清单。

---

## Pending Items

### S-01 文件访问边界收口

- 状态：`Pending`
- 优先级：高
- 涉及范围：
  - `app/api/knowledge_base.py`
  - `app/services/document_service.py`
- 当前问题：
  - 文档预览接口按传入路径读取文件，边界约束不够集中。
  - 删除、重命名、预览虽然已有部分校验，但入口层与服务层仍建议统一收口到 `data/` 目录白名单。
- 建议处理：
  - 所有文档操作统一改为基于内部文档 ID 或相对路径。
  - API 层不再接收任意绝对路径。

### S-02 配置写入与敏感信息保护

- 状态：`Pending`
- 优先级：高
- 涉及范围：
  - `app/services/settings_service.py`
  - `app/api/settings.py`
- 当前问题：
  - 配置中心可直接写 `.env`。
  - 敏感字段虽然有脱敏展示需求，但整体仍缺少更明确的写入审计与权限约束。
- 建议处理：
  - 增加敏感配置修改审计。
  - 区分“读取配置”“编辑配置”“写入密钥”的权限。

### S-03 认证与权限体系缺失

- 状态：`Pending`
- 优先级：高
- 涉及范围：
  - 整体应用
- 当前问题：
  - 当前系统默认更适合本地单用户使用。
  - 如果进入团队共享场景，缺少登录、角色、权限隔离。
- 建议处理：
  - 最小化接入登录态。
  - 提供管理员、普通用户、只读用户等基础角色。

### S-04 CORS 与部署默认值偏宽

- 状态：`Pending`
- 优先级：中
- 涉及范围：
  - `app/config.py`
  - `.env.example`
  - `app/server.py`
- 当前问题：
  - 默认 `CORS_ORIGINS=*` 更适合开发阶段，不适合多人部署。
- 建议处理：
  - 提供开发/生产两套默认值。
  - 部署文档中明确跨域最小开放策略。

### S-05 上传文件校验不足

- 状态：`Pending`
- 优先级：中
- 涉及范围：
  - `app/api/knowledge_base.py`
  - `app/services/document_service.py`
- 当前问题：
  - 当前主要校验扩展名，仍可继续加强大小、内容类型、异常文件处理策略。
- 建议处理：
  - 增加文件大小上限。
  - 增加 MIME 辅助校验与异常文件隔离策略。

### S-06 日志与错误信息脱敏

- 状态：`Pending`
- 优先级：中
- 涉及范围：
  - `app/logging_config.py`
  - `app/api/logs.py`
  - 各 service 的异常日志
- 当前问题：
  - 后续如果引入更多外部连接和多人使用，日志中可能出现敏感路径、请求参数、密钥痕迹。
- 建议处理：
  - 对密钥、Token、内部路径等字段做统一脱敏。
  - 区分开发日志与生产日志。

### S-07 依赖与供应链治理

- 状态：`Pending`
- 优先级：中
- 涉及范围：
  - `requirements.txt`
  - `frontend/package.json`
- 当前问题：
  - 依赖版本当前可运行，但缺少持续性的漏洞扫描、升级策略与锁定规范。
- 建议处理：
  - 引入依赖扫描。
  - 明确前后端依赖升级窗口与回归验证流程。

### S-08 速率限制与资源保护

- 状态：`Pending`
- 优先级：中
- 涉及范围：
  - 聊天接口
  - 上传接口
  - 重建知识库接口
- 当前问题：
  - 当前缺少限流、任务并发保护和资源占用上限控制。
- 建议处理：
  - 为高消耗接口增加限流与并发控制。
  - 防止重复重建、恶意上传或高频问答压垮服务。

---

## 建议回收顺序

1. 文件访问边界收口
2. 配置与敏感信息保护
3. 认证与权限体系
4. CORS 与部署默认值
5. 上传校验与日志脱敏
6. 依赖治理与限流保护
