# 数据库测试与 SQL 校验

本文整理数据库层测试重点、SQL 校验思路和数据一致性检查方法。

## 1. 数据库测试关注什么

重点关注：
- 表结构是否符合设计
- 约束是否生效
- SQL 是否正确
- 数据是否一致
- 更新删除是否有副作用

## 2. 常见数据库测试场景

建议覆盖：
- 插入成功
- 更新成功
- 删除成功
- 唯一索引冲突
- 外键约束
- 事务回滚
- 并发更新
- 软删除过滤

## 3. SQL 校验思路

常用检查：
- 查询数量是否正确
- 核心字段值是否正确
- 关联表是否同步更新
- 更新时间是否变更
- 状态字段是否符合规则

## 4. 慢 SQL 风险

常见风险来源：
- 全表扫描
- 缺失索引
- 模糊查询过重
- 大分页
- 多表 join 过多

## 5. 测试建议

- 关键接口要配合数据库验证
- 不只看接口响应，也要看库里结果
- 数据一致性问题优先级通常很高
