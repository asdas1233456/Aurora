# 安全测试基础检查

本文整理业务测试中最常见的安全风险点，适合功能测试人员做基础安全检查。

## 1. 身份认证

建议检查：
- 未登录能否访问受限资源
- token 过期是否失效
- 退出登录后 token 是否还能用
- 弱口令是否被允许

## 2. 权限控制

建议检查：
- 普通用户是否能访问管理员功能
- 是否能越权查看他人数据
- 是否能越权修改他人数据
- 批量接口是否有权限绕过

## 3. 输入安全

建议关注：
- SQL 注入
- XSS
- 命令注入
- 路径穿越
- 文件上传恶意内容

## 4. 敏感数据

建议关注：
- 密码是否明文传输
- 日志是否打印敏感信息
- 返回体是否泄露手机号、证件号、token
- 导出文件是否包含不该导出的字段

## 5. 基础建议

- 功能测试阶段至少覆盖身份、权限、输入和敏感信息四类风险
- 高风险系统要引入专项安全测试
