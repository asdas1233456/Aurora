# 可访问性测试指南

本文整理 Web 和后台系统中常见的可访问性问题，适合测试在页面验收和回归时快速检查。

## 1. 为什么要做可访问性测试

可访问性不是锦上添花，它影响：
- 键盘用户
- 屏幕阅读器用户
- 低视力用户
- 色弱色盲用户

## 2. 常见检查项

建议检查：
- 仅键盘能否完成核心流程
- 焦点是否可见
- 图片是否有替代文本
- 表单是否有关联 label
- 错误提示是否可感知
- 颜色对比度是否足够

## 3. 表单重点

关注：
- 输入框 label 是否明确
- 必填项是否明确
- 错误提示是否和字段关联
- tab 顺序是否合理

## 4. 按钮和链接

关注：
- 名称是否清晰
- 图标按钮是否有可读说明
- “点这里”这类模糊文案是否过多

## 5. 实战建议

- 核心流程至少手工走一遍键盘操作
- 页面改版时同步做可访问性回归
