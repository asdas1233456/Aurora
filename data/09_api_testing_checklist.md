# 接口测试检查清单

本文适合作为 API 测试执行前的快速检查单，帮助测试人员避免只测 happy path。

## 1. 基础正确性

检查项：
- 请求方法是否正确
- 路径和版本是否正确
- 必填参数是否校验
- 可选参数缺失时默认值是否合理
- 返回状态码是否符合约定
- 返回结构字段是否完整

## 2. 参数校验

建议覆盖：
- 缺失参数
- 空字符串
- null 值
- 超长字符串
- 枚举非法值
- 数字边界值
- 日期格式错误
- 特殊字符

## 3. 鉴权与权限

建议覆盖：
- 未登录访问
- token 过期
- token 伪造
- 无权限角色访问
- 越权访问他人数据
- 接口是否泄露敏感字段

## 4. 业务规则

建议覆盖：
- 重复提交
- 幂等接口多次请求
- 状态流转是否合法
- 资源不存在时是否正确提示
- 已删除资源再次操作
- 非法业务前置条件

## 5. 数据一致性

建议覆盖：
- 创建后是否真正入库
- 更新后查询结果是否正确
- 删除后是否还能查到
- 多表数据是否一致
- 回滚是否生效

## 6. 异常处理

建议覆盖：
- 下游依赖超时
- 下游返回异常
- 数据库异常
- 并发冲突
- 服务重试后的行为

## 7. 接口质量细节

建议关注：
- 错误信息是否可读
- message / code 是否稳定
- 响应字段命名是否一致
- 分页字段是否规范
- 排序规则是否明确

## 8. 常见输出物

接口测试记录建议包含：
- 接口名称
- 测试目标
- 请求参数
- 预期结果
- 实际结果
- 关联日志/截图
- 缺陷编号

## 9. 接口测试建议

- 不要只断言 HTTP 200
- 不要只测单条正常数据
- 接口用例要和业务规则绑定
- 关键接口要沉淀自动化回归
